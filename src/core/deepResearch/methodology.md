# methodology.md

## Phase 1: SCOPE
Analyze the user's request. Determine the boundaries, key entities, and expected outcome. Classify the user intent and required depth (Quick, Standard, Deep, UltraDeep).

## Phase 2: PLAN
Identify the knowledge gaps. Formulate 3-5 macro questions and 5-10 specific search vectors. Define sources of truth (e.g. credible domains, research papers, news, documentation).

## Phase 3: RETRIEVE
Execute web searches or access knowledge bases. Extract raw text, data points, and verify the credibility of the sources. For each piece of evidence, assign a credibility score.

## Phase 4: TRIANGULATE
Cross-verify claims. If Source A and Source B conflict, log the discrepancy. Look for consensus vs outliers.

## Phase 4.5: OUTLINE REFINEMENT
Update the dynamic outline based on retrieved evidence. Determine the 4-8 core findings that will make up the main analysis. 

## Phase 5: SYNTHESIZE
Group the findings into patterns. Determine implications. Draft the initial prose ensuring that >80% is readable text and bullets are used sparingly.

## Phase 6: CRITIQUE
Pass the drafted text through a self-critical lens. Are there any unsupported claims? Are the citations directly mapped to the evidence? 

## Phase 7: REFINE
Fix any issues found during the critique. Replace placeholders with actual citations. Polish formatting and tone for enterprise-grade consumption.
